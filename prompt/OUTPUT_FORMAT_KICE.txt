<work_process>
아래 절차를 순서대로 따르라.

1단계: 구절별 기출 대조
- <target>의 각 구절 S1, S2, ... 를 하나씩 꺼낸다.
- 각 구절의 핵심 표현(2~8어절)이 <references>의 ref 1~N 중 어디에 글자 그대로 등장하는지 찾는다.
- 등장하면 → "기출에 있음" → 수정 금지.
- 등장하지 않으면 → 2단계로 진행.

2단계: 대체 표현 검색 (1단계에서 "기출에 없음"인 구절만)
- 같은 의미를 표현하는 다른 구절이 <references> 어딘가에 있는지 찾는다.
- 찾았으면 → 해당 ref 번호와 해당 구절을 기록하고 수정한다.
- 못 찾았으면 → 수정하지 않는다.

3단계: 자기검증
각 수정에 대해 아래를 확인한다:
(a) revised가 ref[source_index]에 글자 그대로 포함되는가? → 아니면 수정 취소.
(b) original이 이미 어떤 ref에 글자 그대로 있는가? → 있으면 수정 취소.
(c) 수학적 내용(숫자, 조건)이 변경되었는가? → 변경되면 수정 취소.

4단계: 결과 출력
</work_process>

<output_format>
반드시 아래 JSON만 출력하라. JSON 외 텍스트 금지.
{
  "has_edits": boolean,
  "rewritten_full": "수정된 문항 전문 (수정 없으면 원문 그대로)",
  "edits": [
    {
      "source_index": "(정수) revised가 글자 그대로 포함된 ref 번호 (1~N)",
      "original": "원문에서 수정 대상 구절",
      "revised": "수정된 구절",
      "evidence_quote": "ref[source_index]에서 revised를 포함하는 문맥 (15~40자, 가능할 때만)",
      "reason": "수정 이유 (간결하게)"
    }
  ]
}

[자기 점검]
1. original이 <references>의 어떤 ref에 글자 그대로 있으면 → 이 edit 삭제.
2. revised가 <references>의 어떤 ref에도 글자 그대로 없으면 → 이 edit 삭제.
</output_format>
