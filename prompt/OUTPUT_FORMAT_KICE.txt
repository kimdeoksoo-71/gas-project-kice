<work_process>
아래 절차를 반드시 따라 검토하라:

1단계: 구절별 대조
- <target>에 S1, S2, S3, ... 로 번호가 매겨진 구절이 있다.
- 각 구절을 하나씩 꺼내어, <references>의 모든 기출문항과 단어·표현 단위로 대조하라.
- 모든 구절을 빠짐없이 점검하라.

2단계: 수정 판단
- 기출과 다른 부분만 수정. 수정 방향: "기출에 없는 표현 → 기출에 있는 표현"
- 기출에 있는 표현은 절대 수정 금지. 수정할 것 없으면 억지 수정 금지.

3단계: 결과 출력
</work_process>

<output_format>
반드시 아래 JSON만 출력하라. JSON 외 텍스트 금지.
{
  "has_edits": boolean,
  "rewritten_full": "수정된 문항 전문 (수정 없으면 원문 그대로)",
  "edits": [
    {
      "source_index": revised가 글자 그대로 포함된 기출문항 번호(정수, 1~N),
      "original": "원문에서 수정 대상 구절",
      "revised": "수정된 구절",
      "reason": "수정 이유"
    }
  ]
}

[자기 점검]
1. original이 기출에 이미 있으면 → 수정 취소.
2. 수정 방향: "기출에 없는 → 기출에 있는"만 허용.
</output_format>
